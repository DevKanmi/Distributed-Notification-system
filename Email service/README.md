## Description

Email Service for the Distributed Notification System. This microservice handles email notifications by:
- Consuming messages from RabbitMQ email queue
- Fetching user data from User Service
- Fetching and rendering templates from Template Service
- Sending emails via SMTP
- Publishing status updates back to RabbitMQ
- Implementing circuit breakers, retry logic, and idempotency

## Features

- **RabbitMQ Integration**: Consumes email notifications from queue with DLQ support
- **Circuit Breakers**: Protects against cascading failures for User Service, Template Service, and SMTP
- **Retry Logic**: Exponential backoff retry (3 attempts) before moving to DLQ
- **Idempotency**: Prevents duplicate email sends using Redis
- **Template Caching**: Redis caching for rendered templates with automatic invalidation
- **Status Publishing**: Publishes notification status (pending/delivered/failed) to RabbitMQ
- **Health Checks**: Comprehensive health endpoint checking all dependencies
- **User Preferences**: Respects user email notification preferences

```bash
$ npm install

# Edit .env with your configuration
```

## Environment Variables

See `.env.example` for all required environment variables:
- `RABBITMQ_URL`: RabbitMQ connection string
- `EMAIL_QUEUE`: Queue name for email notifications
- `REDIS_HOST`, `REDIS_PORT`: Redis connection details
- `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`: SMTP configuration
- `USER_SERVICE_URL`: User Service base URL
- `TEMPLATE_SERVICE_URL`: Template Service base URL
